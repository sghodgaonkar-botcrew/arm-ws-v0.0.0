## 1. Switch to fixed-size, pre-allocated buffers

In **`ik_model.h`**, change all dynamic-size Eigen/Pinocchio data to fixed 6-DOF types and add member caches:

```diff
--- ik_model.h
+++ ik_model.h
@@ class IKModel {
-  pinocchio::Model     model_;
-  pinocchio::Data      data_;
+  // Use compile-time 6-DOF model for maximal speed
+  pinocchio::ModelTpl<double,6,6>    model_;
+  pinocchio::DataTpl<double,6,6>     data_;
```

## 3. Turn off full Hessian in IPOPT

Computing a 6×6 Hessian every iteration is costly. Switch IPOPT to limited-memory L-BFGS or Gauss-Newton:

In **`ik_solver_v2.cpp`**, when you configure the solver:

```cpp
// Before solve():
options.set_string_value("hessian_approximation", "limited-memory");
// Or, to use Gauss-Newton (JᵀJ) only:
options.set_string_value("hessian_approximation", "exact");
options.set_string_value("nlp_scaling_method","none");
// And skip your get_nlp_hessian callback:

```

Alternatively, implement `get_hessian` as `H ≈ J_cache_.transpose()*J_cache_` (cheap 6×6 multiply) rather than true second derivatives.

## 6. (Optional) Custom Gauss-Newton instead of IPOPT

For a 6 × 6 problem, a tiny Gauss-Newton loop with 2–3 steps and LLᵀ solves can be faster than IPOPT:

```cpp
Eigen::LLT<Matrix6> llt;
for(int iter=0; iter<3; ++iter){
  model.computeJacobian(q);
  Vector6 e = model.error6(q);
  Matrix6 JtJ = model.J_cache_.transpose()*model.J_cache_;
  Vector6 g   = model.J_cache_.transpose()*e;
  llt.compute(JtJ);
  q -= llt.solve(g);
  if(e.norm()<1e-6) break;
}
```

This avoids NLP overhead entirely and often finishes in a few hundred microseconds.
